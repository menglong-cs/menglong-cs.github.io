Tokenizer

tokenizer的作用是将文本切分成token，token可以是一个单词、单个字符或多个字符组成的subword。每个token对应一个整数，所以文本就可以转换成向量输入神经网络。Transformer中最常用的tokenizer是BPE（bype-pair encoding），wordpiece和unigram。

wordpiece和bpe一开始都是从小的vocabulary开始（每个都是单个字符），然后通过与其他字符合并来查看频率和似然概率来进行合并。

其中bpe直接对比a、b两项合并的频率，而wordpiece还需要除于a、b两项单独出现的频率乘积来做归一化。

bpe是根据merge规则，从单个字符一步步进行合并，而wordpiece每次是直接找最长能够匹配的subword。

unigram是从大的vocabulary（所有subword的组合）开始，每一轮去排除一些subword组合，方法是利用unigram语言模型，对比排除哪个subword可以使loss增加的最少。

使用BPE的模型：GPT、GPT2、RoBERTa、BART、DeBERTa等

使用WordPiece的模型：BERT、DistilBERT

Unigram：ALBERT、T5，mBART、BigBird、XLNET

Word Piece

  • 获得足够大的语料库。
  • 定义所需的子词词汇量。
  • 将单词拆分为字符序列。
  • 用文本中的所有字符初始化词汇表。
  • 根据词汇建立语言模型。
  • 通过将当前词汇表中的两个单元组合以将词汇表增加一个来生成新的子词单元。 从所有可能的组合中选择加入词表后能最大程度地增加语言模型在训练数据上的似然概率的组合。（a、b的频率/(a的频率 * b的频率)）
    • $logP(z)-(logP(x) + logP(y)) = log(\frac{P(z)}{P(x)P(y)})$
  • 重复第5步，直到达到子词词汇量(在第2步中定义)，或者似然性增加降至某个阈值以下。

BPE

  • 获得足够大的语料库。
  • 定义所需的子词词汇量。
  • 将单词拆分为字符序列，并附加一个特殊的标记，分别显示单词的开头或单词的结尾。
  • 用文本中的所有字符初始化词汇表。
  • 计算文本中的序列对及其频率。 例如，(‘t’，’h’)具有频率X，(‘h’，’e’)具有频率Y。
  • 根据最频繁出现的序列对生成一个新的子词。 例如，如果(‘t’，’h’)在该对对中具有最高的频率，则新的子字单元将变为’th’。
  • 从第3步开始重复，直到达到子词词汇量(在第2步中定义)或下一个最高频率对为1。在示例中，语料库中的“t”，“h”将替换为“th”，再次计算，最频繁的对再次获得，并再次合并。

BPE是一种贪婪的确定性算法，不能提供多个细分。 也就是说，对于给定的文本标记化文本始终是相同的。

Uni-gram Language Model（ULM）

和WordPiece一样，也是利用语言模型进行subword vocab的选择。

不同之处在于，BPE和WordPiece算法的词表大小都是从小到大变化，属于增量法。而Unigram Language Model则是减量法,即先初始化一个大词表，根据评估准则不断丢弃词表，直到满足限定条件。ULM算法考虑了句子的不同分词可能，因而能够输出带概率的多个子词分段。

-
初始时，建立一个足够大的词表。一般，可用语料中的所有字符加上常见的子字符串初始化词表，也可以通过BPE算法初始化。

  • 针对当前词表，用EM算法求解每个子词在语料上的概率。
  • 对于每个子词，计算当该子词被从词表中移除时，总的loss降低了多少，记为该子词的loss。
  • 将子词按照loss大小进行排序，丢弃一定比例loss最小的子词(比如20%)，保留下来的子词生成新的词表。这里需要注意的是，单字符不能被丢弃，这是为了避免OOV情况。
  • 重复步骤2到4，直到词表大小减少到设定范围。

3.Transformer & GPT & BERT

Transformer为什么使用多头注意力

在超高维空间中做点积来获得向量之间的接近性，意义非常小，因为向量非常分散。分成多个head以后，每个head的embedding维数降低。比如，GPT-3是96头， 这样每个头只有128维。这样利用向量点积计算向量之间的接近性就有效多了。

将隐状态向量分成多个头，形成多个子语义空间，可以让模型去关注不同维度语义空间的信息。（论文原话）

每个注意力头，只关注最终输出序列中一个子空间，互相独立。其核心思想在于，抽取到更加丰富的特征信息。

Attention为什么Q和K使用不同的矩阵生成，而不是使用K和K的点积？

在attention中，主要分为两部分，一部分是相似度矩阵的计算，也就是softmax(kq^T^)，另一部分是利用相似度矩阵将原v值映射到一个新的空间。

那接下来我们需要考虑一点:在已知输入x的情况下，如何建立相似度矩阵？最简单的方式就是自身点乘也就是xx^T^，其可以表征在本空间内的相似度。但通常原空间的相似度是不够的，为了使得模型有更强的表征能力，我们需要其在其他空间映射的相似度，这一点可以参考核函数的设计思想。这时一个最标准的做法就是xWx^T^(参考马氏距离)。如果对其进行拆解就是xW1(xW2)^T^，可以看到此时kq已经有所不同了。W1，W2如果保持相同，那么这个距离是对称自反的，也就是d(x1，x2)=d(x2，x1)。但是在attention中，这种对称自反其实是不必要的，想象一下“我是一个男孩”这句话，男孩对修饰我的重要性应该要高于我修饰男孩的重要性。

Attention中Q和K为什么是点积而不是加法？

3.1 Pre Norm和Post Norm的区别

PreNorm:

$x_{t+1} = x_t+F(norm(x_t)) = x_{t-1} + F(norm(x_t)) + F(norm(x_{t - 1}))$

$= x_0 +F(norm(x_t)) + F(norm(x_{t-1})) + … +F(norm(x_0))$

PostNorm:

$x_{t+1} = norm(x_t + F(x_t))$

post-norm在残差之后做归一化，对参数正则化的效果更强，进而模型的鲁棒性也会更好

pre-norm相对于post-norm，因为有一部分参数直接加在了后面，不需要对这部分参数进行正则化，正好可以防止模型的梯度爆炸或者梯度消失，训练更稳定

Post-norm: 训练不稳定，但若成功收敛，性能比pre-norm好

Pre-norm：训练更稳定，但理论层数不如Post-norm

层数少选post-norm，层数多选pre-norm

3.2 self-attention为什么要除根号下d_k

$attention(Q,K,V)=softmax(\frac{QK^T}{\sqrt{d}}V)$

  • 防止QK^T^的值过大，若QK的值过大，那么经过softmax后
  • 使得QK^T^的结果满足期望为0，方差为1的分布，类似于归一化

3.3 self-attn的计算复杂度

计算复杂度为O(n^2 * d) n是序列长度，d是每个token的维度

Q * K^T^ = [n, d] * [d, n] = [n, n] (复杂度为O(n^2^ * d))

(Q * K^T^) * V = [n, n] * [n, d] = [n, d] (复杂度为 O(n^2^ * d))

4.激活函数

为什么使用激活函数

因为神经网络中每一层都是线性求和的过程，如果没有激活函数，那么神经网络只能拟合线性映射，无法解决更复杂的问题。加入了激活函数，就给神经网络引入了非线性元素，从而使得神经网络能够拟合任何非线性映射，从而解决更复杂的问题。

Sigmoid激活函数

f(x) = sigmoid(x) = 1/(1 + e^-x^)

  • 输出值在(0, 1)之间，相当于对每个输出进行了归一化
  • 可以用于预测概率作为输出的模型
  • 函数是可微的

缺点：

  • 梯度消失：当Sigmod的输入（神经元的输出）较大或较小，梯度接近于0，导致该神经元无法得到更新
  • 计算成本高
  • 输出不以0为中心，输出恒大于0。非零中心化的输出会使得其后一层的神经元的输入发生偏置偏移（Bias Shift），并进一步使得梯度下降的收敛速度变慢。

Tanh 激活函数

f(x) = tanh(x) = (e^x^ - e^-x^) / (e^x^ + e^-x^) = 2 / (1 + e^-2x^) - 1 = 2 * sigmoid(2x) - 1

  • 输出值在(-1,1)之间，输出以0为中心

缺点：

  • 梯度消失，当输入较大或较小时，梯度解决于0，导致神经元无法得到更新
  • 计算成本高

Softmax激活函数

Softmax 是用于多类分类问题的激活函数。

Softmax(x) = xi / (x0 + … + xn)

缺点：

  • 在零点不可导
  • 负输入的梯度为0，因此会产生永不激活的死亡神经元。

Relu 激活函数

f(x) = relu(x) = max(0, x)

  • Rleu是较为流行的激活函数
  • 当输入为正时，梯度为1，一定程度上改善了梯度消失问题，加速梯度下降的收敛速度
  • 计算成本低
  • 符合生物学，具有单侧抑制、宽兴奋边界（兴奋程度非常高）

缺点：

  • 当输入为负时，梯度为0。若参数有一次不恰当的更新，那么第一个隐藏层中的某个ReLU 神经元在所有的训练数据上都不能被激活，那么这个神经元自身参数的梯度永远都会是0，在以后的训练过程中永远不能被激活。

Leaky Relu 激活函数

leaky_relu(x) = $$ f(x)=\left{ \begin{aligned} &x, & if(x > 0) \ &kx, &if(x <= 0) \end{aligned} \right. $$ = max(x, kx)

其中k是一个很小的数，例如0.1或0.01等

  • 扩大了Relu值的范围，负无穷到正无穷

  • 解决了Relu导致参数无法更新的问题，当输入为负数时，梯度为k

尽管Leaky ReLU具备 ReLU 激活函数的所有特征（如计算高效、快速收敛、在正区域内不会饱和），但并不能完全证明在实际操作中Leaky ReLU 总是比 ReLU 更好。

PRelu激活函数

PRelu和Leaky Relu类似，但是当x < 0 时的导数k是可学习的（初始值为0.25）

ELU激活函数

$$ f(x)=\left{ \begin{aligned} &x, & if(x > 0) \ & \alpha(e^x - 1), &if(x <= 0) \end{aligned} \right. $$

$\alpha$是超参数，默认为1

缺点：计算量大，原点不可导

5.损失函数

损失函数就是用来度量模型的预测值f(x)与真实值Y的差异程度的运算函数，它是一个非负实值函数。

损失函数使用主要是在模型的训练阶段，每个批次的训练数据送入模型后，通过前向传播输出预测值，然后损失函数会计算出预测值和真实值之间的差异值，也就是损失值。得到损失值之后，模型通过反向传播去更新各个参数，来降低真实值与预测值之间的损失，使得模型生成的预测值往真实值方向靠拢，从而达到学习的目的。

MSE损失函数

MSE(y|f(x)) = $\frac{1}{n}$ $\sum_{i=1}^n$ (yi - f(xi))^2^

在回归问题中，均方误差损失函数用于度量样本点到回归曲线的距离，通过最小化平方损失使样本点可以更好地拟合回归曲线。

L2损失函数

L2(y|f(x)) = $\sqrt{\frac{1}{n} \sum_{i=1}^n (y_i - f(x_i))^2}$

L1损失函数

L1(y|f(x)) = $\sum_{i=1}^{n} |y_i - f(x_i)|$

交叉熵损失函数

L(y|f(x)) = $-\sum_{i=1}^{n} y_ilog(f(x_i))|$

用于评估当前训练得到的概率分布与真实分布的差异情况

交叉熵损失函数刻画了实际输出概率与期望输出概率之间的相似度，也就是交叉熵的值越小，两个概率分布就越接近

KL散度

KL(y|f(x)) = $\sum_{i=1}^{n} y_ilog(\frac{y_i}{f(x_i)})|$

KL(P||Q) = $\sum{p(x)\frac{p(x)}{q(x)}}$

KL散度也被称为相对熵，是一种非对称度量方法，常用于度量两个概率分布之间的距离。

KL散度也可以衡量两个随机分布之间的距离，两个随机分布的相似度越高的，它们的KL散度越小，当两个随机分布的差别增大时，它们的KL散度也会增大

KL散度是恒大于等于0的。当且仅当两分布相同时，相对熵等于0。

JS散度

用于衡量两个概率分布之间的相似度，它是基于KL散度的一种变形，消除了KL散度非对称的问题，与KL散度相比，它使得相似度判别更加准确。

M = (P + Q) / 2

JS(P||Q) = 1/2 KL(P||M) + 1/2 KL(Q||M)

当两个分布完全不重合时，JS散度是一个常数（log2），梯度为0

7.1 BatchNorm和LayerNorm的区别

Norm的目的是为了将数据拉回标准正态分布，让训练更稳定。

BatchNorm是拿一个Batch的每一维特征进行归一化，LayerNorm是拿每个样本的所有特征进行归一化。

BN抹杀了不同特征之间的大小关系（将Batch内每一维特征进行归一化），保留了不同样本间的大小关系

LN抹杀了不同样本之间的大小关系（将每个样本的所有特征进行归一化），保留了不同特征间的大小关系。

因为每个序列的长度通常不一样（虽然有padding，但无意义），所以无法做batchnorm。

L2范数

L2 = $\sqrt{\frac{1}{n} \sum_{i=1}^n f(x_i)^2}$

L1范数

L1 = $\sum_{i=1}^{n} |f(x)_i|$

Adapter tuning

  • 基于特征的微调是得到模型的输出后进入特定任务的网络层。
  • 全量微调是为每个任务调整所有参数。
  • adapter微调是在网络的一些层中加入adapter，然后为每个任务仅微调adapter的参数。

adapter微调是将adapter注入到原始网络，原始网络的权重不变，而adapter的参数随机初始化。面对一个新任务时，固定原始网络的参数，只需要为每个任务微调adapter的参数。

如果初始化时，adapter导致输入和输出相差太多时，可能导致模型无法训练。所以应该把adapter初始化解决恒等函数。(因为adapter使用的残差连接，所以初始化将adapter的参数设置为接近0)

只需要微调adapter的参数（3.6%），就能达到和全量微调相似的效果。

adapter应用于Transformer Block的attention和FFN层后面（后来有人改进只在FFN层和LayerNorm后面加Adapter模块）

adapter的结构是：现将输入（维度d）映射到低维（m)，然后进行非线性变化，再映射到原始维度（d），所以一个adapter的参数是 (md + d) + (md + m) = 2md + m + d。adapter输入和输出也使用残差连接。

Prefix-tuning

在prompt前面添加一些virtual tokens，在微调期间，只训练前缀的参数（embedding）而固定LLM的参数，从而能够在只微调0.1%的参数就能获得比较好的性能。

在Transfromer的每一层中，prefix的hidden矩阵就是virtual tokens的词向量矩阵（微调的参数为每一层的prefix tokens的hidden矩阵）

训练时，若只微调prefix tokens的hidden矩阵的可能导致训练不稳定并且性能下降，所以在prefix hidden矩阵后面再加个MLP，一旦训练完成，可以将MLP去掉，只保留训练好的prefix tokens embedding作为prefix的hidden矩阵。

prefix使用真实单词的词向量来初始化优于使用随机向量。使用任务相关的词向量由于任务无关的词向量。

若prefix token超过一定数量后，性能会显著下降，可能是拥有更多特殊token会导致输入分布进一步远离预训练数据分布。

优点：

  • 在低数据设置下，表现较好

  • 可以为利用每个用户的数据为每个用户训练prefix，从而避免数据污染。

  • Prefix-tuning可以在一个batch中处理不同的任务（只需要在不同的任务数据前加特定于该任务的前缀即可）

缺点：

  • 占用可用的序列长度。

P-tuning v2

LORA

LORA：冻结预训练模型的权重，将可训练的秩分解矩阵注入到Transformer的每一层。

因为在微调期间预训练好了秩分解矩阵，所以在推理时就可以将它的权重与预训练模型的权重相加，从而降低推理的延迟。

$h = W_0x + \Delta{W}x = W_0x + BAx$ ，其中A是随机高斯初始化，B初始化为0

将Lora只应用在注意力层来减少训练参数，甚至可以只用在query和value权重矩阵上来减少训练参数。

给定训练参数预算，将Lora应用到那些层最有利于模型的性能？

训练参数有限时，只用在注意力层(Q,K,V,O)，参数更有限，只用在query和value

在训练参数有限时，在更多的权重矩阵上使用Lora比提高Lora的秩r更有效。

将Lora的秩r设置为多少是最优的？

Lora的秩非常小的情况下就很有竞争力了。r设置为{1,2,4,8}就足够了。

$W$和$\Delta{W}$的联系有多大？

$W$和$\Delta{W}$具有强相关性。低秩适应矩阵可能会放大特定下游任务的重要特征，这些特征是在一般预训练模型中学习但未强调的。

Lora和Adapter tuning的区别：

  • Lora可以将微调后学习的权重加到原始模型权重上，从而不会带来推理延迟。

  • Lora模块中有两个FFN（没有bias）而Adapter tuning模块中的两个FFN有bias

机器翻译

解码策略

standard greedy search：挑选概率最高的词去输出

beam search：每次挑选概率最高的k个词，最后选择最终概率最大的一条链

sampling：按照概率分布进行采样，让生成的文本更多样化

sampling with temperature：这种方法可以对当前的概率分布进行缩放，例如让概率大的更大、让小的变的更小，或者让大概率和小概率之间差别没那么明显等。

  • 当𝑇变大时，模型在生成文本时更倾向于比较少见的词汇。𝑇越大，重新缩放后的分布就越接近均匀采样。
  • 当𝑇变小时，模型在生成文本时更倾向于常见的词。𝑇越大，重新缩放后的分布就越接近我们最开始提到的贪婪生成方法（即总是去选择概率最高的那个词）。

top-k sampling：只从前概率前top k的词中进行采样

top-p sampling：只从概率大于p的词中进行采样

Speculative Decoding（投机采样）：使用和目标模型结构相似的小模型进行自回归解码，解码n个然后和 输入拼接后输入大模型获取logits以及概率，若大模型在该位置生成该token的概率大于小模型，那么就可以采用小模型解码的结果。

LLM 训练 & 推理

Continue Pretraining和SFT

Continue Pretraining是在领域数据上进行预训练，使用的是没有标签的数据，数据格式和预训练的一样。Continue Pretraining的目的是让LLM更好地处理和理解领域数据

SFT是使用特定任务的数据集，有标签。SFT的目的是优化LLM在特定任务上的表现。

如何解决继续预训练和微调阶段的灾难性遗忘问题

目前不少开源模型在通用领域具有不错的效果，但由于缺乏领域数据，往往在一些垂直领域中表现不理想，这时就需要增量预训练和微调等方法来提高模型的领域能力。

但在领域数据增量预训练或微调时，很容易出现灾难性遗忘现象，也就是学会了垂直领域知识，但忘记了通用领域知识。

解决方法：

  • 调整继续预训练时的数据配比，增加通用数据的比例，增加数据量
  • 使用adapter微调，对于每个任务微调一个adapter
  • 使用正则化的办法可以降低对预训练特征的破坏

LLM训练、推理如何省显存

训练

  • 使用fp16，bf16等低精度来进行训练
  • 梯度检查点：删除中间变量，需要的时候再重新计算
  • ZERO系列优化，参数/梯度/优化器状态等分片存储，存到内存中，即用即取
  • 减小batch size，使用梯度累积

如何缓解大模型幻觉问题

  • 高级提示词，带有详细说明的复杂提示词能够缓解幻觉问题
  • RAG，检索外部知识库
  • 验证链：
    • 生成基线响应：给定一个查询，使用 LLM 生成响应。
    • 设计验证：给定查询和基线响应，生成验证问题列表，有助于自我分析原始响应中是否存在任何错误。
    • 执行验证：依次回答每个验证问题，然后将答案与原始响应进行检查，以检查是否存在不一致或错误。
    • 生成最终验证响应：鉴于发现的不一致（如果有），生成包含验证结果的修订响应。
